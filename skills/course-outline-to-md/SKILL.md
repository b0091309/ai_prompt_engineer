# Course Outline To Md

將課程大綱草稿轉換為清晰、結構化的 Markdown 檔案，優化內容以適合影片演說和教學簡報使用。

## 處理流程

### 1. 分析草稿內容

首先理解草稿中的核心元素：
- 課程主題與目標
- 章節劃分與順序
- 每個章節的重點內容
- 教學要點或說明事項

### 2. 結構化組織

將草稿內容轉換為層次分明的 Markdown 結構：

**基本結構範例：**

```markdown
# 課程名稱

## 課程簡介
[課程概述、目標、適合對象]

## 第一章：章節標題
### 本章重點
- 重點 1
- 重點 2

### 教學內容
[詳細說明]

### 補充說明
[額外資訊或注意事項]

## 第二章：章節標題
...
```

### 3. 優化演說友善性

針對影片演說進行優化：

**層次清晰**
- 使用清楚的標題層級（#, ##, ###）
- 每個段落聚焦單一概念
- 適當使用列表呈現要點

**視覺引導**
- 使用粗體強調關鍵概念：**重要術語**
- 適當使用引言區塊突出重點：
  ```markdown
  > 💡 關鍵提示：這是最重要的概念
  ```

**節奏控制**
- 將長段落拆分為易於口述的短段
- 使用分隔線（---）標示主題轉換
- 在複雜內容前加入「接下來我們來看...」等引導語

### 4. 內容增強

根據草稿內容適當補充：

**架構元素**
- 如果草稿缺少課程簡介，補充概述性段落
- 為每章節加入簡短的引言
- 在章節結尾加入小結或過渡語

**教學輔助**
- 使用編號列表呈現步驟流程
- 使用項目符號列表呈現並列概念
- 適當使用程式碼區塊展示範例（如適用）

**視覺標記**
- 使用 emoji 標記不同類型的內容（可選）：
  - 📚 理論知識
  - 💡 重點提示
  - ⚠️ 注意事項
  - ✅ 實作練習

## 輸出格式

建立完整的 Markdown 檔案，包含：

1. **檔案開頭**：課程名稱作為主標題
2. **目錄結構**（選擇性）：如果章節較多，可加入目錄
3. **主要內容**：結構化的章節內容
4. **檔案結尾**：課程總結或延伸資源（如適用）

## 實例示範

**輸入草稿範例：**
```
Python 基礎課程
第一週 變數和資料型態 講解基本概念
第二週 條件判斷 if/else 語法 練習題
第三週 迴圈 for while 應用
```

**輸出 Markdown 範例：**
```markdown
# Python 基礎課程

## 課程簡介
本課程為期三週，帶領初學者掌握 Python 程式設計的核心概念，從變數操作到流程控制，建立紮實的程式基礎。

---

## 第一週：變數與資料型態

### 本週重點
- 認識 Python 的基本資料型態
- 學習變數的宣告與使用
- 理解型態轉換的概念

### 教學內容
介紹 Python 中的基本概念，包括整數、浮點數、字串等資料型態，以及如何正確使用變數來儲存和操作資料。

---

## 第二週：條件判斷

### 本週重點
- 掌握 if/else 語法結構
- 學習條件表達式的撰寫
- 透過練習題強化理解

### 教學內容
深入探討條件判斷的運作原理，學習如何使用 if、elif、else 來控制程式流程，並透過實際練習題加深印象。

---

## 第三週：迴圈控制

### 本週重點
- 理解 for 迴圈的使用時機
- 掌握 while 迴圈的語法
- 學習迴圈在實際問題中的應用

### 教學內容
學習使用 for 和 while 迴圈來重複執行程式碼，探討不同迴圈結構的適用情境，並透過實際案例演練應用技巧。

---

## 課程總結
完成本課程後，您將具備 Python 程式設計的基礎能力，可以撰寫簡單的程式來解決日常問題。
```

## 注意事項

- 保持草稿原意，不過度添加未提及的內容
- 確保 Markdown 語法正確，便於後續編輯和渲染
- 文字流暢自然，適合口語化演說
- 標題層級不超過三層（#, ##, ###），避免結構過於複雜
- 將 Markdown 檔案儲存至 `/mnt/user-data/outputs/` 目錄以便使用者下載
